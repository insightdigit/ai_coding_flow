# Bug 修復流程

## 觸發條件

當用戶提問包含以下關鍵字時自動啟用：
- 「bug」「錯誤」「問題」「修復」「fix」「壞掉」

## 執行流程

```
1. Sequential Thinking - 分析問題根因
2. /impact-analysis - 評估修改影響範圍
3. /tdd - 撰寫測試重現 Bug
4. coder agent - 修復問題（讓測試通過）
5. reviewer agent - 審查
```

## 步驟說明

### 1. 分析問題

- 現象：Bug 的具體表現？
- 預期行為：正確行為應該是什麼？
- 重現步驟：如何重現？
- 根因假設：可能的原因？

### 2. TDD 測試重現

撰寫失敗測試來重現 Bug，確認修復後測試通過。

### 3. 修復原則

- 最小修改：只改必要的程式碼
- 避免副作用：不引入新問題

## 品質要求

- **NEVER** 不寫測試就修復 Bug
- **ALWAYS** 執行完整測試確認無回歸
