# 架構設計流程

## 觸發條件

當用戶提問包含以下關鍵字時自動啟用：
- 「架構」「設計」「評估」「比較」「方案」

## 執行流程

```
1. Sequential Thinking - 深度分析架構選項
2. 產出方案比較與建議
3. reviewer agent - 架構審查
```

## 分析項目

- 需求理解：要解決什麼問題？
- 約束條件：效能、成本、時間限制？
- 方案列舉：有哪些可能的解決方案？
- 方案比較：各方案的優缺點？
- 風險評估：各方案的風險？
- 建議方案：推薦哪個方案？為什麼？

## 輸出格式

```markdown
## 架構分析

### 需求摘要
[簡述需求]

### 方案比較
| 方案 | 優點 | 缺點 | 適用場景 |
|-----|------|------|---------|
| A | ... | ... | ... |
| B | ... | ... | ... |

### 建議
[推薦方案及理由]
```

## 品質要求

- 架構必須符合專案現有模式
- 必須考慮可擴展性
- 必須經過架構審查
